FR:-

1. User should be able to reserve a parking spot.
2. System should generate a ticket/receipt.
3. 3 Types of vehicles[Compact, Regular, Large]
4. Price is Flat rate based on time but different rate for each vehicle type.
5. User must not be able to make the payment.
6. Send Notification to user when parking spot is booked/canceled

NFR :-

1. Low Latency
2. High Consistency [User must not be able to book the same parking spot]
3. Third Party system Like Stripe can take care of payment.

10-Floor Building with 2000 parking spot/garage

API Design

|-----------------------------------------------------------------------------------------------
| POST    |   /reserve           |  (garageId, startTime, endTime)  => Ticket (spotId ,reservationId)
| POST    |   /payment           |  reservationId --> Third Party system Like Stripe
| POST    |   /cancel            |  reservationId
| ###########    Internal EndPoints     #########################
| GET     |   /calculatePayment  |  reservationId
| PUT     |   /freeSpot          |  (garageId, vehicleType, time)
| PUT     |   /allocateSpot      |  (garageId, vehicleType, time)
| POST    |   /create_account    |  User user
|-----------------------------------------------------------------------------------------------
